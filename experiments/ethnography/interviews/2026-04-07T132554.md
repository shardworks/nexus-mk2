## By Experiment

### X008 — Patron's Hands

**Pipeline confidence is infrastructure-attributed, not evidence-accumulated.** Sean explicitly
linked the review drop-off to the planning pipeline and revision rig, not to a run of successful
outcomes. The session notes corroborate: reviewed_at data shows the cliff at April 4 coincides
with those milestones, not with an uptick in success rates. This is a direct H4 signal.

**"Raising the height of mountains" as the next stress test.** Sean has a conscious plan to
push the pipeline harder — bigger briefs, more ambiguous scope — and see where trust recalibrates.
This is a deliberate, methodologically aware approach to H4/H5. He's not waiting for failure to
recalibrate; he's planning to provoke it.

**The D11 drop and the pipeline-not-me response.** When a patron override was silently dropped,
Sean's response was to route the fix to the pipeline, not to increase his own spec-reading
vigilance. This is a strong signal about where he sees the accountability boundary. He's
confident enough in the broader pipeline that one failure doesn't trigger behavioral regression.

**Review as memory prompt.** "I meant to do it when we first implemented role instructions — I
just forgot until that review." The commission review is playing a role beyond quality assurance:
it surfaces deferred intentions. This is an unanticipated X008 signal — the review layer has
a mnemonic function that isn't captured in H3 or H4.

**Complexity-8 evaluation challenge.** "In order to really test this one, I would need a whole
rig design that needed the feature." This is a new capability ceiling for functional evaluation:
some commissions can't be assessed through use without implementing dependent features first.
Sean fell back to code review as proxy. The review infrastructure (scorer + Coco) becomes more
load-bearing as commission complexity grows.

### X013 — Commission Outcomes

**w-mnnmd63t probed.** Complexity 8, partial/broken. Key findings:

Sean has patched it at the source (artificer role instructions). This is a notable
reclassification: what looked like multiple partial outcomes was actually a single systemic gap.

## Cross-Cutting Observations

**The planning pipeline has crossed a threshold.** "Working so well our current workstream is
building a framework-native version" — this is not incremental adoption. The pipeline is being
promoted from sanctum tooling to a shippable framework feature. The patron experience has

work for complexity-8 commissions. These incidents are functioning as diagnostic readouts
for concerns that weren't surfaced in normal operation. The review layer — even when reading

is important: Sean's current trust level is explicitly provisional. He's planning to probe it
deliberately, not waiting to be surprised. This is a sophisticated relationship with the
system's calibration.
