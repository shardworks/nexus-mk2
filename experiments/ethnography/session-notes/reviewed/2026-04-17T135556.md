# Session Summary

## What we did

- Sean pointed at session f5b4e148 (post-skill-patch) where Coco still looked inefficient on click tooling. I pulled the transcript and diagnosed the real pattern: `extract` *was* used on the parent subtree — the skill promotion worked — but without `--full`, so only goals came back. The two follow-up `show` calls weren't show-walking; they were a workaround for an extract that dropped the conclusion payload.
- Per Sean's direction, patched `packages/plugins/ratchet/src/tools/click-extract.ts` in the framework to always include conclusions — removed the `--full` flag entirely, no opt-out. Library API (`ratchet.extract`) unchanged. Updated the obsolete goals-only test, the CLI tool table in `ratchet/README.md`, and the `--full` reference in `.claude/skills/clicks/SKILL.md`. Framework commit **8ed3c17** (rebased), sanctum commit **ab6cc64**.
- Investigated Sean's separate `nsg writ show` silent-output concern. Found two distinct issues: (1) writ tools don't support short-id resolution at all (unlike all click tools, which call `ratchet.resolveId`); (2) the "silent" transcript behavior was actually `2>/dev/null` masking a "Writ not found" error.
- Fixed both. Framework commit **9da1e70** added `ClerkApi.resolveId` (exact-match fast path + LIKE prefix scan with ambiguity detection) and wired it through every id-accepting writ tool (`writ-show`, `writ-edit`, `writ-complete`, `writ-fail`, `writ-cancel`, `writ-publish`, `writ-link`, `writ-unlink`, `piece-add`, `commission-post`). Added 4 new `resolveId()` tests + 1 adjusted regex on piece-add. Smoke-verified `nsg writ show w-mo2xi6pt` now returns the writ.
- Sanctum commit **607f989** added an agent-prompt directive: don't reflexively use `2>/dev/null` — filter stdout (`| head`, `| grep`) to cut noise, leave stderr alone so real errors stay visible.
- Concluded **c-mo2xtsqp** with the finding that the extract-always-full fix makes the proposed `show`-ancestry augmentation unnecessary.

## Decisions made and/or deferred

- **Decided:** `click extract` always includes conclusions at the CLI — no opt-out flag. Library default stays `full=false`. Rationale: the CLI is used almost exclusively for orientation, where conclusions are the payload that matters.

- **Decided:** `show`-ancestry CLI augmentation is unnecessary. Concluding c-mo2xtsqp now that the root cause (extract dropping conclusions) is fixed.
