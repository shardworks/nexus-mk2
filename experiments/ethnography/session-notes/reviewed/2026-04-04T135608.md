# Session Summary

## What we did

- **Diagnosed plan pipeline session forking failure** — agents running in temp codex clones couldn't `--resume` sessions because Claude stores sessions per directory path. Fixed by using stable temp dir names based on session ID (`c55e58b`).
- **Diagnosed and fixed plan pipeline agent role abandonment** — reader agent was ignoring its inventory role and trying to implement the brief directly. Root cause: `--agent` flag layers custom instructions on top of Claude Code's default "helpful coding assistant" system prompt, which competed with specialized role constraints. Fixed by switching to `--system-prompt-file` + explicit `--tools`, `--setting-sources user`, `--permission-mode acceptEdits` (`11edca9`).
- **Added transcript capture** — pipeline now copies session JSONL to `specs/{slug}/planner-transcripts/` before deleting temp clones.
- **Gated auto-chain on output** — analyst no longer auto-starts unless `inventory.md` actually exists.
- **Moved agent prompts** from `.claude/agents/plan-*.md` to `bin/plan-prompts/` — no longer agent definitions, just system prompt files.
- **Created X003 experiment artifact** documenting all failure modes and the fix (`b5abce8`).

## Decisions made and/or deferred

- **`--system-prompt-file` over `--agent`** — for specialized pipeline agents, replacement beats addition. The default system prompt actively undermines constrained roles.
- **`--bare` rejected** — breaks OAuth/keychain auth. `--setting-sources user` achieves the CLAUDE.md suppression without killing auth.
- **Hardcoded output paths** — dropped the `{specs_dir}` placeholder pattern. Concrete paths in user prompts are more reliable than template substitution.
- **Deferred: guild integration for plan workshop** — Sean's long-term vision is real animas with Books/tools. Current MCP-less approach works for now; custom write tool or Books integration is the next evolution.
