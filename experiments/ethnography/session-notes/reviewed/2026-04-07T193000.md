## Decisions made and/or deferred

- **Hono** as web framework — zero deps, built-in TS, selected over Express (65 deps), Fastify (42 deps, competing plugin model), Koa (30 deps + separate router).
- **Pages only, not widgets** — plugins contribute full pages as static asset directories. No cross-plugin layout coordination.
- **CLI-mirror REST naming** (`/api/writ/list`) over RESTful (`/api/writs`) — mechanical transform, zero ambiguity, one mental model across CLI and web.
- **`'cli'` caller type renamed to `'patron'`** — eliminates need for a new `'web'` caller type. Both CLI and web are patron-facing interfaces.
- **HTTP method from permission level** — `read` → GET, `write`/`admin` → POST, `delete` → DELETE.
- **Server-side chrome injection** — Oculus injects nav and stylesheet into page `index.html` responses.
- **Shared CSS, three layers** — custom properties, element-type selectors, utility classes. Tokyo Night dark palette.
- **The Oculus** as the name (not Astrolabe) — Astrolabe reserved for a future planning-workflow apparatus.
