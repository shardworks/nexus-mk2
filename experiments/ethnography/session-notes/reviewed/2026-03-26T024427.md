## Decisions made and/or deferred

- **`npm install @latest` over `npm update`** — Sean asked whether package.json fields would change. `npm update` only touches the lockfile; `npm install foo@latest` updates both the specifier in package.json and the lockfile. Went with `@latest`.
- **Version stamp moved to CLI** — `applyUpgrade()` in core no longer stamps the version. The CLI does it unconditionally after both npm updates and bundle application, so guilds get a current version even on npm-only upgrades.
- **`--dry-run` skips npm update entirely** — dry-run is read-only; no npm side effects. Could add `npm outdated` reporting later but deferred.
