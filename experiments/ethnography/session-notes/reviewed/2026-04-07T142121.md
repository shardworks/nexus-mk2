## Decisions made and/or deferred

- **Choice answers use discriminated objects** (`{ selected: string } | { custom: string }`) not bare strings — prevents ambiguity between typo'd option keys and intentional custom answers.
- **`allowCustom` flag on choice questions** — mirrors planning workshop's need for write-in answers.
- **Two-phase completion** — answering questions doesn't unblock; patron explicitly calls `input-complete` or `input-reject`.
- **Rejected status triggers checker failure** — cascades engine → rig → writ failure through existing infrastructure.
- **Engine resume via book query** (`rigId + engineId + status:completed`, most recent) rather than `priorBlock` — survives process restarts, handles multi-block naturally.
- **`rigId` must be added to `EngineRunContext`** — folded into the patron-input commission since it didn't make it into the checker-failure one.
- **Brief goes through planning pipeline** rather than using the spec candidate directly — the spec candidate was built conversationally and has gaps the planner will catch.
- **CDC-driven unblocking deferred** — 10s polling is the mechanism; CDC is a future optimization for all block types.
