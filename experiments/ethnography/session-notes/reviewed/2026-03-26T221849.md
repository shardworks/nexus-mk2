# Session Summary

## What we did

- **Designed the commission removal architecture** — traced all commission/writ touch points across core, stdlib engines, CLI, and MCP tools. Identified the commission/mandate duality as the root cause of sync bugs.
- **Discovered the workspace disconnect** — the `mandate.ready` event payload lacks `worktreePath`, meaning artificers dispatched via the new standing orders run in the guildhall, not their worktrees.
- **Designed writ enrichment** — new fields: `workshop` (nullable), `source_type` (`patron|anima|engine`), `source_id` (nullable).
- **Drafted spec** at `.scratch/spec-remove-commissions.md` covering schema migration, core changes, engine rewrites, new `nsg writ` CLI, tool changes, and event flow redesign.

## Decisions made and/or deferred

- **Commissions fully removed** — no deprecation path, clean break. Mandate writ type also retired.
- **Workshop nullable on writs** — present = managed worktree lifecycle + full tools. Null = no worktree, restricted (read-only) tool set enforced by summon-engine.
- **CLI requires explicit choice** — `--workshop <name>` or `--no-workshop`. Omitting both is an error.
- **Workshop inherited by children** — `createWrit` copies parent's workshop when omitted.
- **Source tracks record creator** — `sourceType: patron|anima|engine`, `sourceId` nullable.
- **Deferred:** multi-workshop writs, planning anima, pull-based session dispatch.
