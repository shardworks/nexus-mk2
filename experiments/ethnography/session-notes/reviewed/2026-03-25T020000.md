## Decisions made and/or deferred

- **Static precondition types for now, custom JS later** — `command`, `command-output`, `env`. Schema reserves room for `custom` type. Sean noted arbitrary code concern is moot since `command` already runs arbitrary shell commands.
- **Roles are config + files, not installable packages** — role instructions are living documents edited in place (like codex), not versioned/packaged artifacts (like curricula). Optimizes for the common case: editing instructions frequently.
- **Role instructions are their own thing** — not curricula (different lifecycle: live vs snapshotted) and not codex (scoped to role, not guild-wide). Path relative to guild root.
- **Hard error on assigning undefined role; soft warning at manifest time** — protects instantiation integrity while allowing guild.json edits without breaking running animas.
- **`baseImplements` replaces `["*"]`** — bundle-installed and default implements go here.
- **Deferred:** `custom` precondition check type, role management CLI commands (`nsg role create/add-implement`), docs updates.

### Notable moments

- **"i'd like to support flexible guild structures where tools don't assume the presence or names of specific roles"** — Sean reframing roles from implicit strings to first-class structural positions. This decouples implement authoring from guild organization.

- **"i'm wondering if anima should have a list of roles, or if roles should have a list of anima"** — Sean's instinct that the guild's structure (roles as positions) should own membership, not the other way around. Led to `seats` capacity model and the conceptual flip where roles are organizational positions that animas fill.

- **"think deeply about how people are going to use this system once it gets traction"** — Sean pushing back on over-engineering roles as installable packages. Led to the simpler "config + files" design after analyzing day-to-day user workflows. Notable for grounding architectural decisions in actual usage patterns rather than type-system elegance.

- **"note that 'instructions' and 'content' are relative to their installed path, not to the guild root"** — Sean catching a consistency gap that nearly led to roles becoming a full artifact type before the usage analysis redirected the design.
