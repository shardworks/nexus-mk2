## Decisions made and/or deferred

- **Async spawn:** Use async `child_process.spawn` for Claude CLI, not `spawnSync`
- **No interactive teardown:** Temp worktrees from interactive sessions are NOT auto-deleted (data loss prevention). Reaping is out of scope.
- **Untyped transcripts:** `Record<string, unknown>[]` for now, type later when analysis needs arise
- **Session ID on result:** `SessionResult` includes `sessionId` so clockworks can write `commission_sessions` join rows
- **Error handling:** try/finally guarantees around provider launch; `session.record-failed` core event for failures
- **UUID file naming:** Session records use `crypto.randomUUID()` filenames
- **Merged phases:** Plan Phase 1+2 merged into single phase (avoids homeless `generateMcpConfig`)
- **Fold engines into core:** worktree-setup and ledger-migrate absorbed into core (not stdlib) since they're fundamental capabilities other CLIs/tools will need
