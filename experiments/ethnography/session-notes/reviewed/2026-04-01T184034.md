# Session Summary

## What we did

- **Created `bin/coco-all.sh`** — a new script that runs one-shot `claude -p` sessions across all packages in the nexus monorepo (`/workspace/nexus/packages`). It discovers packages via `package.json`, expands a `${PACKAGE_PATH}` template variable, passes a system prompt file via `--system-prompt-file`, and batches concurrent sessions (default 8). Includes progress output and a pass/fail summary.
- **Added `--agent` flag** — made the agent configurable (`-a`/`--agent`), defaulting to `coco`.

## Decisions made and/or deferred

- **System prompt via file, not inline** — used `--system-prompt-file` to avoid shell-escaping large markdown documents.
- **`--add-dir` per package** — each invocation gets `--add-dir <pkg>` so the agent has file access to the target package.
- **Output suppressed** — claude stdout/stderr goes to `/dev/null` since the agents do their work via side effects (commits, file writes). Could add `--output-dir` later if capture is needed.
- **CWD is caller's CWD** — the script doesn't `cd` anywhere; the invoking directory is preserved.

## Next steps & open questions

- **Try it out** — run the script against a real system prompt and see how it behaves at scale.
- **Output capture?** — might want a `--output-dir` flag to save per-package output to files for review.
- **Filtering?** — could add a `--filter` or `--package` flag to target a subset of packages.
