## Decisions made and/or deferred

- **Vision authoring on disk**: `<GUILD>/vision/<slug>/{vision.md, vision-metadata.yml}`; patron-editable, file → writ one-way flow.
- **Cartograph keeps vision writ** as canonical snapshot; no `product` writ type introduced; charges still parent on vision.
- **CLI verb = `apply`**: `nsg vision apply <slug>` is a single code path for first-time bootstrap and Nth re-import.
- **Sidecar identity**: `visionId` only — no slug field. Slug is a directory-name affordance for CLI ergonomics, not a durable identifier.
- **Apparatus boundary**: cartograph + vision CLI's scope ends at "writ is a current snapshot." Decomposition, charge reconciliation, and rig dispatch belong to the **Surveyor apparatus** (designed elsewhere).
- **Stale-binding behavior**: warn at scan, error on apply attempt; patron explicitly transitions writ to sunset/cancelled. No auto-cascade.
- **"Feature" vs. "charge"**: `charge` is canonical; "feature writs" was a misspeak.
- **Deferred**: charge reconciliation strategy (Surveyor's call), fs-watcher/git-hook triggers (post-v0), drift detection (flagged for later).
