# Session Summary

## What we did

- **Designed and implemented `@shardworks/instruments`** — a generic LLM instrument runner as a TypeScript package in the sanctum monorepo (`packages/instruments/`). 8 source modules, 30 tests. Handles config parsing, shell-based extractor execution, template expansion, parallel `claude --print` calls, structured YAML output parsing, and statistical aggregation. (`9cfe43a`)
- **Created two independent instruments** — `spec-blind-quality-scorer/v1` and `spec-aware-quality-scorer/v1` at `experiments/instruments/`, replacing the old mode-based approach. Each is fully self-contained with extractors, prompts, and instrument.yaml.
- **Built instrument resolution CLI** — supports explicit path, name+version, or name-only with auto-version-select. Default instrument root: `experiments/instruments/`.
- **Ran full regression test** — all 22 scored commissions re-run through new runner. 13/22 identical composites, 9 with small deltas (±0.08 to -0.50). Analyzed the 4 widest outliers dimension-by-dimension — all are single-dimension integer flips on borderline scores, consistent with expected instrument variance.
- **Retargeted Laboratory CDC trigger** to call `quality-review-v2.sh` instead of old `quality-review-full.sh`. (`cd17725`)
- **Validated assembled prompt equivalence** — 11 commissions with saved context compared; differences are only minor whitespace from a double-fence bug in the old bash runner.

## Decisions made and/or deferred

- **Shell extractors + TypeScript runner** (hybrid) — extractors stay shell (git plumbing), runner is TS (config parsing, output validation, aggregation).
- **No modes — two instruments instead.** Blind and aware are independent instruments, not variants. Simpler runner, instruments can evolve independently.
- **Instruments elevated to `experiments/instruments/`** — span across experiments, not X013-specific.
- **Extractor duplication over shared dir** — each instrument is fully self-contained for reproducibility.
- **Deferred:** QS-1 (5-point scale), QS-2 (structured concerns), deleting old scripts, CL-1 through CL-5.
