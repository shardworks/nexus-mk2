## Decisions made and/or deferred

- **Rollup functions split into check + complete** — `checkXCompletion` (read-only) and `completeXIfReady` (mutates). Engines decide policy; functions don't auto-complete on mixed results (some children failed).
- **Pieces stay `active` on child failure** — when a job fails, its parent piece stays active until manually resolved. No `failed` status for pieces/works (schema unchanged).
- **`completeXIfReady` are engine-only** — no MCP tools for them. Animas use `*-update` for manual status changes.
- **Artificer gets only `job-check`** — higher-level rollup checks are steward oversight.
- **Doc location: `docs/reference/`** — three files (core-api, event-catalog, schema). Not architecture docs, not guides — reference material.
- **Deferred: doc delivery mechanism** — Phase 4 assesses how to get docs to animas working outside the monorepo, once we know doc size.
- **Deferred: `dispatch-list` and `audit-list` as MCP tools** — operator-only via CLI for now.
