- **Drafted spec** at `.scratch/external-mcp-servers.md` covering: `guild.json` schema (`servers`, `baseServers`, `roles.*.servers`), manifest engine changes (`resolveServers()`, `ResolvedServer` type, env var interpolation), availability checks, system prompt integration for unavailable servers, and output format mapping to Claude Code's `.mcp.json`.

## Decisions made and/or deferred

- **Servers are config, not artifacts.** No descriptors, no install command, no `nexus-tool.json` equivalent. You edit `guild.json` directly. Rationale: external MCP servers are infrastructure declarations ("this guild uses GitHub"), not authored packages.
- **Role gating via `baseServers` + `roles.*.servers`.** Same pattern as tools. Gives fine-grained control without new concepts.
- **Instructions for external tools live in role/codex docs**, not on the server entry. Server config is pure launch config; teaching is the guild's domain.
- **Env var interpolation with `${VAR}` syntax.** Keeps secrets out of `guild.json`.
- **Deferred:** individual tool gating within a server (needs proxy layer), SSE/HTTP transport, dedicated install/remove commands, server health monitoring.
