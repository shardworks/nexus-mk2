## Decisions made

- **Triggers (MVP):** A-level blocked transitions (stuck/failed/retry-cap-exhausted) + queue-drained. Skip completion-side (B7 needs-review flag has no mechanism to hook).
- **Source of truth:** writ-state CDC. Reckoner observes; queue-drained derived in-stream on each terminal transition.
- **Channels (MVP):** Discord webhook (push) + CLI inbox (query/fallback). Lattice has a channel abstraction for future additions.
- **Pulse shape:** immutable event record. Fields: `id, source, trigger_type, writ_id, title, summary, link_url, context, created_at, delivery_state`.
- **Response surface:** read-only. CLI verbs `list` / `list --live` / `show`. No ack/dismiss/resolve.
- **Rate-limit:** nothing for MVP. Principle: rate-limit and digest logic belong on the Lattice (delivery side), not the Reckoner (emission side).

Explicit "chose NOT to":
- **No ack system** — the writ is source of truth for "still live"; shadow state on pulses duplicates it.
- **No Herald carrier role** — Lattice delivers directly.
- **No Discord interactive buttons** — requires a full Discord bot, not just a webhook.
- **Deferred:** digest, quiet hours, patron-present suppression, multi-recipient, additional channels, rig-level events.
