## Decisions made and/or deferred

- **Guild monitor over publishing site** — monitor exercises the framework's read APIs, is immediately useful, and has distributed complexity. Publishing site deferred as second project.
- **Separate workshop repo, not monorepo package** — keeps agent-built code out of the framework. Agent exports a function; Sean wires the CLI integration.
- **Commission spec stripped to essentials** — function signature and dev harness prescribed (consuming contract), layout/content left to the agent.
- **NFRs in codex, not commission** — TypeScript, licensing, dependency maturity are guild-wide standards, not per-commission.
- **Stroke enforcement deferred** — fix curriculum first, then codex, then framework directive, then enforcement. Escalation ladder, not immediate framework injection.

### Notable moments

- **Sean lost sight of the big picture:** "I'm losing sight of the big picture of what I'm doing." Coco reframed: the system exists to dispatch agents and evaluate outputs, but no real commission had been dispatched yet. "There's a very well-documented orchard" with no fruit.
- **Sean on framework isolation:** "I don't want agents operating on the core framework itself" — surfaced the practical constraint that the only codebase was the framework, which agents shouldn't touch. Led to the guild-monitor project.
- **Commission spec pushback:** Sean: "It's a lot more detailed than I was expecting" — caught Coco writing a blueprint instead of pointing at a mountain. Led to stripping layout prescriptions, directly testing the project's own philosophy.
- **Stroke non-adherence discovery:** Six explicit directives in the system prompt, zero usage. Sean: "Does this mean we need to somehow inject this at a framework level, so that the guilds don't have to think about it (or get it wrong)?" — surfaced the instruction vs. enforcement design question.
- **Curriculum as root cause:** Sean: "I wonder how much of that is because it had the full guild ops guide from the steward in its head" — identified that the agent used CLI commands because the curriculum taught it about the CLI, conflating the patron's interface with its own.
- **Codex insight:** Sean noted the codex has a unique property: "It allows the operator of a guild to apply instructions to anything they build — wherever they are running the AI agents, and whatever repositories they are building in. It's got a unique stickiness and global nature."
