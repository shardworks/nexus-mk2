# Session Summary

## What we did

- **Built `nsg upgrade` command** — full upgrade path for guilds: `planUpgrade()` diffs bundle migrations and content against guild state, `applyUpgrade()` executes the plan. CLI: `nsg upgrade [--dry-run] [--recompose] [--bundle <spec>]`. Core module at `packages/core/src/upgrade.ts`, CLI at `packages/cli/src/commands/upgrade.ts`. 9 tests. Commit `8b286c2`.
- **Added stale anima detection** — `checkAnimaStaleness()` and `checkAllAnimaStaleness()` in `anima.ts` compare composition versions against on-disk training content. Surfaced in `nsg anima list`, `nsg anima show`, `nsg status`, and MCP tools `anima-list`/`anima-show`.
- **Added `--recompose` flag** — retires stale animas and recreates them with same name/roles/curriculum/temperament, picking up latest training content. Old anima gets timestamped name for audit trail.
- **Updated steward role instructions** — added "Upgrades & Staleness" section to both `init-guild.ts` (template for new guilds) and the Shardworks guild's `roles/steward.md` directly. Steward now knows how to detect and fix stale animas.
- **Documented known gap** — role instructions aren't in the bundle manifest, so they don't upgrade automatically. Filed at `docs/known-gaps.md`.
- **Wrote dashboard reference** — `.scratch/dashboard-anima-staleness-fields.md` for an artificer commission to add staleness to the guild-monitor UI.

## Decisions made and/or deferred

- **Numeric migration naming kept** over semver-prefixed — provenance metadata in `_migrations` table already tracks which bundle delivered each migration. Naming is redundant.
- **Bundle-diff approach for upgrade** over version-tagged or content-hash alternatives — leverages existing provenance tracking.
- **Option B (retire/recreate) for stale animas** over in-place mutation or composition history — Sean pointed out retire/recreate is nearly free (standing orders are role-based, no commission reassignment needed).
- **Upgrade is CLI-only, not an MCP tool** — it's an operator action (fetches npm packages, applies DDL). Steward can detect staleness and fix animas but cannot trigger upgrades.
- **Deferred:** Role instructions as a bundle artifact category (known gap).
