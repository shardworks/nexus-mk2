### Notable moments

- **Sean reframed the npx scope.** I'd dismissed npx-for-everything
  as too slow, but then dismissed npx-for-bootstrap-only too quickly
  by lumping them. Sean: *"that's how real users will do it. I'm
  open to either, but just wanna make sure we didn't dismiss 'one
  time npx' too hastily."* Reopened the architectural question and
  ended up with a cleaner answer (Option H bootstrap mirrors user
  UX exactly).
- **Sean caught the redundant probe registry.** *"a bit late... but
  i think we should probably NOT have a specific probe registry...
  doesn't the `fabricator` already provide an engine lookup?"* ‚Äî
  forced an architectural simplification. The type-guard pattern is
  better than what I'd written.
- **Sean caught a misnamed field.** *"I also note that
  `trial-context.yaml` has `0.0.0` for the frameworkVersion...
  probably not what we want."* ‚Äî surfaced the lab-host vs. test-guild
  scope confusion in the probe summary. Fix shipped same round.
- **The first real trial caught two real bugs.** Both were the kind
  of thing unit tests had no path to detect (ESM-vs-CJS module
  shape, Spider's variable-substitution mechanic). Validates the
  premise of the apparatus: real trials surface assumptions
  synthetic tests miss.
- **Sean's strict-pinning stance.** *"We probably shouldn't allow
  specifying filesystem version of plugins (including CLI) for
  reproducibility purposes."* ‚Äî clarified the apparatus's
  reproducibility contract: manifests are inputs you can re-run
  later, so every reference must content-address.
