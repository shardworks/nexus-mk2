## Decisions made and/or deferred

- **Named hierarchy over generic tree.** Named levels carry operational semantics the system needs; a generic parent/child tree defers those questions to runtime.
- **Terminology:** work, piece, job, stroke. "Task" retired. "Deed" and "undertaking" rejected after contextual testing. "Job" won for workhorse frequency; "stroke" for craft resonance.
- **Commission = origin, not size.** A commission is the patron's request; it maps onto whatever hierarchy level fits. Separated from the size hierarchy entirely.
- **Strokes tracked in the Ledger.** Not below the system's notice ‚Äî strokes are the progress boundary, the context bridge for staged sessions, and the foundation for capability tiering.
- **Hierarchy is framework-level; roles are guild-level.** The four levels and their semantics are part of Nexus. Which roles perform which operations is guild policy via standing orders.
- **"Work" vs "works" distinction.** "Work" (hierarchy) = body of labor requiring decomposition. "Works" (metaphor) = everything delivered to the patron. Related but distinct.
- **Deferred:** Ledger schema design for hierarchy (parent/child relationships, decomposition tracking). Implementation specs for stroke tool. Architecture docs noted as describing future state.
