## Decisions made

- **"Static engines" are not a thing.** Every static engine was absorbed into core or stdlib because they were just libraries with no shared API. The concept is removed from the guild metaphor doc. Sean noted he could see a plugin-style engine type emerging in the future (e.g. session provider slots), but we don't have that today.

- **Engine = Clockworks event handler.** The guild metaphor doc now defines engines solely as automated processes that respond to Clockworks events via a standard handler contract. Everything else is just a dependency.
