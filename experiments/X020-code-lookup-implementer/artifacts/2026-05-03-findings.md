## Diagnosis

**The simplest version of H1's mechanism ÔÇö "the model invokes code-lookup, code-lookup queries are cheaper than Grep+Read, therefore cost drops" ÔÇö did not fire.** The Ôłĺ20.6% cost gap with 0% adoption rules it out at N=1.

What might explain the cost gap (with the tool unused)?

1. **N=1 session-to-session variance.** Implementer behavior on this brief may have Ôëą20% noise floor. We have no paired re-runs to estimate it.
2. **Partial scope shift.** With-tool produced 3 fewer doc files. At a few percent of the gap; doesn't fully explain it.
3. **Snippet behavioral side-effect.** The snippet was added as a top-level "Tool preference" section in the artificer role file. Its framing ("structural queries via code-lookup, textual via Grep") may have nudged the model toward more focused tool sequences without invoking code-lookup. The 25% drop in total tool calls (155 Ôćĺ 116) and 52% drop in Edit calls (44 Ôćĺ 21) are compatible with a more deliberate session shape.
4. **Implementation-attempt artifacts.** The first attempt failed mid-flow (80 tool calls of progress, then $0/0-token crash). Retry succeeded. Possible the second attempt benefited from some implicit state effect not visible in the data.

**Crucially, X020 weakens ÔÇö not supports ÔÇö the X019 role-mismatch diagnosis.** X019 said: "the planner doesn't ask find-callers questions; the implementer's edit workflow does." X020 put the find-callers tool directly in the implementer's hands, told it explicitly when to use it, and the implementer never reached for it across two attempts and 196 tool calls. The implementer's edit workflow on this brief did not generate find-callers queries either.

The dropBook brief is admittedly a narrow probe ÔÇö most of the implementer's work was *adding* a new method, *implementing* its backends, *writing tests* against it, and *updating docs*. The "before changing X's signature, find every caller" canonical use case wasn't present in this brief; it was a downstream-application brief, not a refactoring brief. A future X020 follow-up on a refactoring-shaped brief might fire the mechanism.
