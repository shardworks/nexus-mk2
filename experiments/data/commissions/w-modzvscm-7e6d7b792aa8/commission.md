Per D7, `nsg clock run` will call `processEvents()` in a loop until it returns `processedEvents: 0`. Two iterations are required when relays themselves emit events (CDC auto-wiring on dispatch-row writes will emit `book.clockworks.event_dispatches.created` events on every dispatch, and any relay that writes to a watched book emits more events). For a guild with a standing order on `book.<x>.<y>.created`, this means one operator `run` could process N rounds before settling. The dispatcher's per-call standing-order revalidation makes this safe; but this commission should ship a test where iteration N+1 picks up an event emitted by iteration N's dispatch, just to lock the behavior in. Worth a 1-test addition rather than a follow-up writ.
