The apparatus-doc template (`docs/architecture/apparatus/_template.md:98-103`) directs authors to "Remove this section when the spec is finalized" but offers no guidance for the common in-between state: an apparatus is past MVP and some (not all) Open Questions have been decided in code. Ratchet is the first case where this primer has had to call out the drift; it's unlikely to be the last.

A lightweight follow-up could be a guild-level norm — e.g., when a code commit resolves an Open Questions bullet, the same commit (or an immediate follow-up) migrates the answer into Implementation Notes. Or an audit-pass commission that sweeps all nine apparatus docs in `docs/architecture/apparatus/*.md` and reconciles each Open Questions list against current code. Either is a separate concern from this brief.
