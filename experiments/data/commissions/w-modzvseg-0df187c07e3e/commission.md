Per D6, `runDispatchSweep` will gain an optional `onDispatch?: (row: EventDispatchDoc) => void` callback. The contract should be explicit: the callback fires immediately after each dispatch row is written, BEFORE the parent event's `processed` flag is flipped to true. This ordering matters for any CLI / observer that wants to render 'event id X has these N dispatches' as a coherent group. Document this in the JSDoc on the new opt; otherwise future callers may assume undefined ordering and write fragile observers.
