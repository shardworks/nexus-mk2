`packages/plugins/clerk/src/writ-type-config.ts` leaves the `name` field rule at "Non-empty. Format rules are enforced elsewhere." The validator checks only non-emptiness. This is fine for the structural module, but the schema-reference doc needs to tell plugin authors what format rules apply: do writ-type names have to be kebab-case? alphanumeric? prefixed with the plugin id like link kinds? Today the Clerk accepts any string and stores it verbatim in `WritDoc.type`. Without explicit guidance, plugin authors will invent conventions and the type vocabulary will end up inconsistent. Either the refactor enforces a format rule (parallel to `KIND_SUFFIX_RE` in clerk.ts for link kinds) and the docs state it, or the docs note that no format rule is enforced and give a recommended convention. Worth settling before the doc ships rather than after.
