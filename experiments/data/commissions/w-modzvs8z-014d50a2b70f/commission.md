`packages/framework/cli/src/program.ts` `buildToolCommand` (line 115) wraps the tool handler in `try { result = await handler(...); console.log(result); } catch { console.error('Error: ' + msg); process.exit(1); }`. There is no path through this contract for a tool that succeeded but should exit nonzero (e.g. `nsg clock tick` with `status: error` dispatches per this commission's brief). Two ways to fix: (a) recognise a structured return shape `{ output: string; exitCode: number }`; (b) document that tools requiring this should be hand-written framework commands. Either is fine; today the CLI silently steers people toward hand-writing. Worth surfacing once a second tool hits this.
