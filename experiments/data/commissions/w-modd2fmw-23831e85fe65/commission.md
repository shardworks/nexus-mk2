Astrolabe already contributes two non-mandate writ types at `packages/plugins/astrolabe/src/astrolabe.ts` lines ~282–289: `piece` and `observation-set`. These are currently just `WritTypeEntry` shells with `name` + `description` — they have no declared state machine and silently inherit mandate's phase machine. Post-T2 every registered type MUST have a full config (states, allowedTransitions, childrenBehavior), so T2's migration must decide: (a) astrolabe authors new full configs for piece and observation-set, (b) T2 provides a default mandate-cloned config for types that don't ship one, or (c) existing WritTypeEntry callers are refactored. This is T2 scope, not T1—but T1 authors should be aware the universe of registered types is already >1 in existing guilds.
