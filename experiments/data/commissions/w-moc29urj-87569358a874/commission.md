The Ratchet's `conclude`, `drop`, `park`, and `resume` methods (`packages/plugins/ratchet/src/ratchet.ts:272-322`) all follow a non-transactional read-then-patch pattern: `await api.get(id)` → validate → `clicks.patch(...)`. Under concurrent callers this has the same TOCTOU race that amend has (and that this commission closes for amend by using `stacks.transaction`). For `conclude`, the race is worse: the `if (click.conclusion !== undefined) throw ...` check and the subsequent `clicks.patch` can both see `undefined` and both write a conclusion, with the second silently overwriting the first under the current code path — a direct violation of the "conclusion is write-once" guarantee at `docs/architecture/apparatus/ratchet.md:357`.

The fix is to adopt the same `stacks.transaction((tx) => { const txClicks = tx.book<ClickDoc>('ratchet', 'clicks'); ... })` wrapper already used by `reparent` at `ratchet.ts:331-357`. Applies to all four methods; a single commission that sweeps them together is the right shape.
