RigDoc schema gap: no rig-level terminal timestamp
The typed RigDoc interface (spider/src/types.ts:93) carries only createdAt; there is no completedAt, endedAt, updatedAt, or terminalAt. D3 works around this by deriving end time from max(engine.completedAt) when the rig status is terminal, but that fallback is fragile in edge cases (a rig that fails before any engine starts, or a cancelled rig whose engines were all pending). A future refactor could add a persisted rig-level terminalAt timestamp set when the rig enters a terminal status. Not in scope for this brief, but worth tracking — the elapsed display will occasionally render <1s for these degenerate terminal rigs.
