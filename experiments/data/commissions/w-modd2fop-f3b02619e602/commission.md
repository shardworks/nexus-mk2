The brief lists the check: "no state with no inbound transitions (unless classified `initial`)." Literal reading: a terminal state with zero inbound transitions also fails. But can that happen in a sensible config? A terminal state with no inbound path is genuinely unreachable (no state can transition to it) — which IS a bug in the config. So the check as written is correct. Capturing this as an observation in case the T1 implementer reads the brief's check and worries about terminal-state edge cases; the intent is: every non-initial state must be reachable from somewhere. Terminal or active doesn't matter; reachability does.
