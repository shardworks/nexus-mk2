If the Reckoner's Phase 2 watcher is invoked multiple times for the same transition (current Stacks semantics don't replay Phase 2 events, but plugin bugs or future changes could), emit() would produce duplicate pulses. A defensive check — either a (writId, prevPhase, newPhase) composite key or a small 'already-emitted' set keyed by the CDC event timestamp — would make the observer idempotent. For MVP, rely on the single-fire guarantee; record as a hardening concern so it's not forgotten when CDC semantics ever change.
