`packages/plugins/spider/src/rig-view.ts` collects (engineId, sessionId) pairs from engine.sessionId only. Under per-attempt history, an engine that retried twice has three sessions total (all recorded in attempts[]). Current rig-view reports only the latest session's cost, understating retry-heavy rigs by ~N×. Implementation should decide: sum across all attempt sessions (most accurate), show only latest (current behaviour), or show both (latest + total). Probably belongs inside this commission's S11 scope, but flagging as observation because the brief's explicit CLI ask focuses on attempt/hold state, not cost.
