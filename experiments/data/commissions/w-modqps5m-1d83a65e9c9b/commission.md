`packages/plugins/clerk/pages/writs/index.html` lines 84-92 hardcodes phase filter buttons `All / new / open / completed / failed / cancelled`. `stuck` is a real mandate phase (active, non-terminal — the 'needs attention' state) and is missing from the filter bar. An operator cannot scope the table to stuck writs without a different mechanism. T5's classification-based filter (D9 selects `classification-buttons`) inherently fixes this gap (`stuck` is `active`), but if the patron prefers an alternative (D9 option B or C) this gap should be addressed there too.
