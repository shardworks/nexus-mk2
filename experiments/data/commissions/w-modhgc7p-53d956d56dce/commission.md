`packages/plugins/clerk/src/clerk.ts` `post()` wraps the parent-existence + parent-phase check in `stacks.transaction(...)` only when `request.parentId` is supplied (L382–421). The non-parented branch (L424–436) writes directly. Post-T2, parent gating becomes classification-driven (per D21), but the underlying race window is unchanged: a parent transitioning to a terminal state between the check and the child write is theoretically possible if the watcher path is bypassed. Today this is a no-op concern (no producer races CDC + post). Worth noting because the strict transition validator (S2) makes the temporal coupling more visible: if the parent goes terminal mid-transaction, the child's write succeeds but the parent's terminal state will eventually bring scrutiny. Not in T2 scope; surface as a candidate for a follow-up commission to either always-wrap or document the contract.
