`loadWrits()` short-circuits to an empty table when `currentType.size === 0` without hitting the API (lines ~902–907 of `packages/plugins/clerk/pages/writs/index.html`). This is pre-existing behavior, but in the deep-descendant world it's worth noting: the hierarchy disappears entirely if the operator deselects every type filter, and they may not realize children are hidden because of that filter rather than the data. A small 'All types deselected — select at least one type to see writs' empty-state message would be clearer. Out of scope for this brief.
