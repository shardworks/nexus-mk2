`reckonings-book.md` §'Validator namespace observation' notes that `reckoner.` and `reckoning.` are NOT in `RESERVED_EVENT_NAMESPACES` in `packages/plugins/clockworks/src/signal-validator.ts`. The doc says this is 'not load-bearing in v0' because the only event the book produces is the auto-wired `book.reckoner.reckonings.created`, and `book.` itself is unreserved. But if a future commission adds named-event emission (`reckoning.accepted` etc.), that commission must reserve the namespaces or animas could forge those events via the `signal` tool. Worth a one-line cross-reference comment in the Reckoner CDC handler when it lands, so the next contributor sees the dependency.
