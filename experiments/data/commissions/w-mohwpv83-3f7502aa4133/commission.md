D20 selects `read-on-each-call` for `disabledSources` so operators can disable a misbehaving petitioner without restarting the guild. This relies on `guild().guildConfig().reckoner` reflecting on-disk edits at every call — same convention Clockworks uses for standingOrders (clockworks.ts line 261). It currently works because `Guild.guildConfig()` returns the in-memory config object and writes go through `guild.writeConfig()`.
