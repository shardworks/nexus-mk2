The D11 semantics (target reachable from every non-terminal state) require one BFS per `childrenBehavior` transition target. With N=6 states and E=6 edges (mandate's scale) this is trivial. For types with 30+ states the cost is still sub-millisecond. Flagging because a naive implementation that runs BFS *inside* the per-state loop (instead of computing the full forward-reachability closure once and then testing membership) would multiply the cost. Not a scaling concern; just a code-review note for T1 implementer ("compute reachability closure once; test targets against it").
