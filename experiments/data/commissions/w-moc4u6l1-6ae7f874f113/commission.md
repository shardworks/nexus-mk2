`packages/framework/cli/src/commands/init.ts` calls `createInitialGuildConfig` (in `packages/framework/core/src/guild-config.ts`) which writes `plugins: []`. The init handler then prints a hint to run `nsg plugin install @shardworks/nexus-stdlib` — a package that does not exist in the workspace.

This means the 'Standard Guild' described in `docs/architecture/index.md` is a documentation concept with no enforcing code path. The doc says (lines 11, 141, 259) that `nsg init` produces / populates the standard guild; it does not. This brief's prose tweak (D10) closes the immediate inconsistency, but the underlying product question remains: should `nsg init` install a baseline (e.g. stacks + clerk + codexes + loom + tools + animator + spider + fabricator + ratchet + claude-code + parlour), and if so, how is that baseline declared in code? A separate commission should pick this up; the answer determines whether `nexus-stdlib` (or an equivalent meta-package) is needed.
