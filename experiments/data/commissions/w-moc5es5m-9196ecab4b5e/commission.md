While reading `setTypeFilter()` in `packages/plugins/clerk/pages/writs/index.html` (lines ~1431–1464) it became apparent that `currentType.size === 0` is only reachable by deliberate user action (toggling off the last selected type). An alternative fix to the ambiguity is to treat an empty set as equivalent to 'All selected' and rebuild the query accordingly — then the writs-table view would never disappear on filter action. This is a UX choice with a different trade-off than the explanatory-message fix this brief chose, and is worth surfacing for the patron's consideration but not worth blocking the current brief.
