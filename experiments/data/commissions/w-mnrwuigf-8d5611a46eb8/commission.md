Followup to `w-mnrroznr-ef81f7ffe972` ("Cancellable Animator Sessions").

We now have the ability to cancel individual anima sessions at the Animator level. We should nextadd the ability to cancel Quick Engines running an anima session. This capability should be surfaced as a Tool, with a specific 'Cancel' button visible on the engine detail of a running quick engine in the Oculus. If the spider already has appropraite states and state transitions for cancelled engines, this should obviously plugin to that. If not, we should design and implement them for this feature. (Generally idea being mark all other unstarted engines as skipped, and any active/blocked engines become cancelled). We should also somehow invalidate any pending 'engine blocks' which are associated with blocked engines which get cancelled by this. The final state of the rig itself should reflect that it was cancelled, as should the writ.
