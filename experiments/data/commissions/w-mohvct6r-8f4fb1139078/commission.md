The Reckoner contract §7 documents that framework-mediated identity (caller pluginId stamped via AsyncLocalStorage proxy) was rejected for the Reckoner specifically because `4+ current plugins` use raw timer / signal / HTTP / IO callbacks that drop ALS context. The same rejection applies to setWritStatus / setWritExt — today both APIs trust the caller to stamp its own pluginId, and a misbehaving plugin can write to another plugin's sub-slot. The identity-proxy work continues at `c-mofxdlwb` (independent track). Out of scope for this commission per D6 (`convention-only`); recorded so the trade-off remains visible if the identity-proxy track ever delivers a usable API.
