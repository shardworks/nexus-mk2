During planning, the system 'lifts' observations from the spec writing phase to new writs to be implemented. However, these observations assume the original writ is already completed and would be confusing, difficult, or impossible to implement first. However, the generated writs do not express this dependency. The 'observation-lift' engine should make sure that when new writs are generated, they use the correct 'dependency' semantic link type from the spider to prevent the generated writs from being scheduled before the original one.
