D3 recommends embedding `writUpdatedAt` inside `pulse.context` and indexing via a query on `context.writUpdatedAt`. This works but mixes dedupe infrastructure into the context payload shape (which is otherwise trigger-type-scoped). A first-class optional `idempotencyKey: string` on `PulseDoc` (indexed) is cleaner, at the cost of a Lattice schema change. The recommendation stays with context-embedded for now because it is emitter-local, but if multiple emitters adopt the pattern, consolidating to a first-class field becomes the right move.
