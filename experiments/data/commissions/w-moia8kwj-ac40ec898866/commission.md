Once this commission registers the `reckoner/reckonings` book via `supportKit.books.reckonings`, `clockworks-stacks-signals` will start auto-emitting `book.reckoner.reckonings.{created,updated,deleted}` Clockworks events for every row write. There is no carve-out (per `reckonings-book.md` §'No carve-out from auto-wiring'). This is the intended channel for petitioner standing orders — but it also means any guild that installs the Reckoner immediately starts producing CDC events at whatever rate the Reckoner emits reckonings. For an early-stage guild this is fine; for a guild approaching the storage-growth ceiling (~1.5M rows/year per `reckonings-book.md`'s math), the event volume on the Clockworks side may eventually warrant attention. Track for retrospective when first production guild hits steady state.
