`packages/plugins/clerk/src/clerk.ts::handleChildTerminal` and `handleParentTerminal` (lines ~769–813) implement only **failure propagation** and **parent-terminal downward cancellation**. There is no path that completes a parent when all children reach `completed` — the parent simply stays in `open` until a caller explicitly completes it. Yet the mandate config that T2 will author almost certainly declares `childrenBehavior.allSuccess: { transition: 'completed' }`, since the T3 brief lists "mandate parent with two child mandates, both complete → parent transitions from `open` to `completed`" as an explicit verify-scenario. This means **T2+T3 together introduce all-success auto-completion for mandates for the first time**, even though the refactor narrative frames both as pure refactors. Any existing live mandates sitting in `open` with all-completed children will, at the moment T2+T3 land, spontaneously transition to `completed`. A migration-awareness / release-note entry is warranted; also consider a one-shot audit query at T2 startup to report how many mandates would be affected so the patron can see the blast radius before the rollout.
