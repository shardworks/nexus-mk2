`registerKitWritTypes()` in `packages/plugins/clerk/src/clerk.ts` validates only that `entry.name` is a string. It does not call `validateWritTypeConfig()` on kit contributions — because kit entries are still `WritTypeEntry`, not `WritTypeConfig`. Once T2/T3 ships the shape change, the kit-registration path needs to invoke `validateWritTypeConfig()` on each contribution and hard-fail on validation errors (matching `registerKitLinkKinds()`'s fail-loud treatment). This is what makes the validator's error-path taxonomy (which the schema reference documents per D7) actually surface to plugin authors. Purely an implementation follow-up, but the docs will claim behaviour that the runtime doesn't yet honour until this wiring lands — observing so the spec writer knows to verify against the right version of clerk.ts.
