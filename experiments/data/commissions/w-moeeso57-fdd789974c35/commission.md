`packages/plugins/claude-code/src/babysitter.ts` `readConfigFromStdin` (around lines 121–156) parses incoming JSON and validates required fields by walking a hand-maintained `(keyof BabysitterConfig)[]` literal array (`['sessionId', 'guildToolUrl', 'dbPath', 'logDir', 'claudeArgs', 'cwd', 'env', 'prompt', 'tools', 'startedAt', 'provider']`). If `BabysitterConfig` ever grows a new required field, the literal array must be updated by hand — TypeScript will not catch the omission because `(keyof T)[]` happily accepts any subset of keys.

The package already depends on `zod` (used in `detached.ts` for `z.toJSONSchema()`), so adding a `BabysitterConfigSchema` and replacing the manual loop with `BabysitterConfigSchema.parse(parsed)` is a low-cost change. Bonus: zod's error messages would be stricter and more useful than the current generic `Missing required config field: <name>`. Tests in `babysitter.test.ts` that exercise rejection cases („throws on missing required fields“, „logDir validation“) would need their expected error messages updated.
