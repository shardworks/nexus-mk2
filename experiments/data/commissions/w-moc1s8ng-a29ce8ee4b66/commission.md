The post form in `packages/plugins/clerk/pages/writs/index.html` (the `#post-section` / `submitPostForm()` block) only creates root writs — there is no `parentId` field or 'Add child writ' button on a detail view. `ClerkApi.post({ parentId })` already supports child creation, and the `commission-post` tool accepts `parentId`. Adding a 'New child' button on the detail view would close the loop so operators can build hierarchies through the UI, not just the CLI. Out of scope for this brief (which is read-only display), but it pairs naturally with the deep-descendant view.
