The Oculus click detail page at `packages/plugins/ratchet/pages/clicks/index.html:623-718` renders full click detail (goal, status, parent, children, links) but has no Amend button and no indicator that a click has prior goal versions. The brief explicitly keeps UI changes out of scope for this commission, but operators using the Oculus page will be unaware both that the affordance exists and that a given click has been amended. A follow-up UI commission could add: (a) an Amend button in the live-click action bar alongside Park/Conclude/Drop at index.html:657-660, wiring to a new POST `/api/click/amend`; (b) a 'Prior goals (N)' disclosure in the Details grid near line 645 that expands to list the amendedAt/goal entries; (c) the same 'has history' hint in the tree row to aid at-a-glance triage.
