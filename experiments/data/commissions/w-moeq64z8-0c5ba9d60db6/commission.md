`packages/plugins/spider/src/types.ts` L261–265 documents `writTitle` as: *'Omitted when the writ cannot be resolved (e.g. deleted).'* But `enrichRigView` (rig-view.ts L52–54) also leaves `writTitle` unset when `rig.writId` is falsy: `const writTitlePromise = rig.writId ? writsBook.get(rig.writId).then(...) : Promise.resolve<string | undefined>(undefined);`. In current Spider operation every rig has a `writId` (it's how the rig was spawned), so the no-writId branch is unreachable in practice. But the docstring claims the omit-case is *only* the deleted/missing-writ case, which is narrower than what the code actually does.

Not a bug — the docstring under-describes a degenerate case that does not arise in production. A small docstring tweak could close the gap: *'Omitted when the writ cannot be resolved (deleted, missing, or rig has no writId).'* Or, alternatively, the rig-view code could fail-loud on a rig with no `writId` (per Three Defaults: prefer fail-loud to silent fallback) since that state is otherwise impossible. Either change is cosmetic; the production behavior is correct.
