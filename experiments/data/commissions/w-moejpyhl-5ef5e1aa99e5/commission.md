Stacks' `MAX_CASCADE_DEPTH = 16` is the only protection against runaway cascade. Today's upward-only engine consumes ~1 depth per parent level (grandparent lift adds 1 more). Adding `parentTerminal` doubles the per-level cost: child fails (depth 0) → parent’s `anyFailure` fires (depth 1) → parent terminal event re-enters → `parentTerminal` cancels remaining children (depth 2) → each child's terminal event may itself fire its own `parentTerminal` cascade onto grandchildren (depth 3+). A balanced tree depth N now consumes roughly 2N depth instead of N. The 16-deep cap gives ~7-deep balanced trees with both directions wired.
