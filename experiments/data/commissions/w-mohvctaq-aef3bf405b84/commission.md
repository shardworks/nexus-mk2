The current `transition()` implementation has an asymmetric strip rule that is surprising to readers: a non-empty `phase` in `fields` throws (`[clerk] transition: cannot override phase via fields argument`), while `status` (and now `ext`) are silently stripped. The asymmetry is correct — phase is Clerk-owned, the slots are plugin-owned — but the rationale is buried in the implementation comment (clerk.ts — lines ~1080–1099) rather than being a doc-level statement. Worth a one-paragraph addition to the clerk.md / README.md transition() section explaining: phase is the one Clerk-owned managed field and rejects smuggled writes; the plugin-owned slots silently strip because the smuggle is a compatibility hazard, not a contract violation.
