D4/D6 above land a small but new design pattern in the codebase: the `animator-status` tool's response is no longer a verbatim mirror of `AnimatorApi.getStatus()` — it adds a server-computed field. This is the first such case in the animator package; most tools (and the prior commission's D20 specifically) have been verbatim-doc returns.

This is fine for the current scope (the predicate truly is a presentation-layer concern), but it establishes a precedent that downstream commissions might extend into less benign cases (computing aggregations, joining other state, etc.). Worth surfacing as a pattern in the animator README's section on the tool, with a clear bound: tools may compute *display-only* derived fields whose source data is in the API's verbatim return; the API itself stays verbatim. Otherwise the tool layer drifts into being a parallel API surface.
