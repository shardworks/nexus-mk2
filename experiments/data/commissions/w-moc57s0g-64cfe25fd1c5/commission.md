The post form markup is inside `#writ-list-view`, so any flow that needs to show the form from the detail view must first toggle the two views (currently handled by `openRepost()`). This coupling means we cannot render the form inline in the detail view without structural HTML refactoring. If future work wants a second entry point from the detail view that feels native (without the jarring list-view swap), consider lifting `#post-section` to be a sibling of both views so either pane can host it. Not in scope for this brief, but it would make the child-creation UX smoother.
