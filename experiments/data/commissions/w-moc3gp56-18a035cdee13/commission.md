When the Oculus writs detail view opens, `showWritDetail` (packages/plugins/clerk/pages/writs/index.html lines 917–939) and `refreshDetail` (lines 873–903) each call `buildDescendantTree(id)` which recurses through `/api/writ/list?parentId=` per level. With the new deep `summary`, that same walk is also being done server-side for the summary counts. A writ-tree endpoint (sibling w-moc1s8sd) or an `items: deep` extension would let both server and client share one traversal instead of two.
