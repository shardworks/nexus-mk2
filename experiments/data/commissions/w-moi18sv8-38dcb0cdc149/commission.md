`packages/plugins/spider/src/engines/draft.ts` line 35 uses `execSync('git rev-parse HEAD', { cwd: draft.path, encoding: 'utf-8' })` directly to read the worktree's HEAD. Every other git invocation in the codexes/spider apparatus goes through the typed `git()` helper in `packages/plugins/codexes/src/git.ts` (returns `GitResult` / throws `GitError`, has 10MB output buffer, is consistent across the package). The draft engine's `execSync` call is a one-off that bypasses the helper and would benefit from being switched to the shared async wrapper for consistency — small refactor, no behavior change. Particularly relevant after this commission, since the new reattach code in scriptorium-core.ts will exercise the same `git rev-parse HEAD` pattern via the typed helper, making the draft engine's hand-rolled call the lone outlier.
