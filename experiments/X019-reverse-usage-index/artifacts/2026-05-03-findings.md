## What the 2 code-lookup calls actually did

- **`code-lookup package=@shardworks/clerk-apparatus`** → 30,057-char return (whole package surface). Followed by a targeted Grep in clerk's source. Use: narrow Grep target, not replace it.
- **`code-lookup symbol=StacksApi`** → 330-char return (file + line + signature). Followed by a precise Read at offset/limit. Use: locate-then-Read.

Neither was a "find all usages" query — the use case where code-lookup structurally beats Grep on token volume.

## Diagnosis: workflow mismatch

The reader-analyst Read **303K bytes** across 37 Reads, with 7 of the top 10 reads being **whole files** (≥13K bytes each). The role's bottleneck is *understanding implementations*, not *finding things*.

- 12 surviving Greps were mostly **implementation-pattern searches** (`setWritExt|getWritExt|writ\.ext`, `nested|already in transaction`, `book\.cartograph|cartograph.*visions`, …). These need contextual matches inside file bodies — code-lookup can't replace them.
- The 13 Greps that were *eliminated* in the with-tool variant got mostly subsumed by **Read** (fetch the file directly), not by code-lookup (structured queries).
- Behavioral side-effect: the snippet's "use Grep only for textual searches" framing nudged the model toward Read-instead-of-Grep, even when code-lookup wasn't reached for.

The mechanism is intact: tool exists, model uses it, results are correct. **The reader-analyst's task domain just doesn't have many "find all usages" queries.** A planner does deep dives; signatures + JSDoc don't replace reading whole implementations.

## Comparison to X018

X018 attempted to put structured data **into** the system prompt; the cache-write cost on each session swamped the orientation savings. X018 failed *on cost*.

X019 attempted to put structured data **behind a tool** (on-demand, no persistent cache cost). On cache economics, this is the right shape — cache reads dropped 10% (8.59M → 7.75M). **But the tool didn't get reached for often enough to convert the cache savings into a 25% win.** The mechanism economics are sound; the role's appetite for them is too low.
