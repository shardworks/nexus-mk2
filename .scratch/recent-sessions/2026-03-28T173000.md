## Decisions made and/or deferred

- **`allowedContexts` was wrong** because "context" already means the runtime injection object (`RigContext`, `ToolContext`) in this codebase — naming collision flagged in code review.
- **`allowedChannels` was an improvement but still off** — "channel" is borrowed vocabulary (communication medium / release channel connotation).
- **`surface` considered and rejected** — Sean found it too abstract.
- **`allowedCallerTypes` considered and rejected** — accurate but too long for config-in-js authoring.
- **`callers` considered** — clean noun but felt like metadata rather than config.
- **`callableFrom` chosen** — reads naturally at the authoring site (`callableFrom: ['cli']`); Sean explicitly prioritized the tool-author config-in-js experience as the deciding lens.
- **Scalar-or-array input decided**: `tool()` input accepts `ToolCaller | ToolCaller[]`, normalized to array on output — Sean's call, reduces friction for the common single-caller case.
