## What we did

- **Audited `book-store.ts` for SQL injection** ‚Äî found two interpolation sites: field keys in `buildWhereClause` and `query.orderBy` in `find()`. Added `validateFieldName()` allowlist guard (`/^[A-Za-z0-9_.-]+$/`) wired into `toJsonPath()` to cover both.
- **Fixed table-name safety** ‚Äî replaced throw-on-unsafe with normalization: `normalizeRigKey()` maps `/` ‚Üí `__` and other non-identifier chars ‚Üí `_`. Removed erroneous "no sanitization needed" comment. Discussed and resolved collision risk (slash uses `__` to preserve structural distinctness from `-`/`_`).
- **Audited `reconcile-books.ts`** ‚Äî found it had its own local `toJsonPath()` with no validation, same injection vector. Exported `validateFieldName()` from `book-store.ts` and wired it in. Also updated stale JSDoc examples to reflect new normalized table names.
- **Fixed `SqliteAdapter` DDL bug** ‚Äî `stmt.columns()` throws for DDL statements; `reconcileBooks` would crash in production on first guild with any rig books declared. Switched to `stmt.reader` boolean which is safe for all statement types.
- **Fixed `OFFSET` without `LIMIT`** ‚Äî SQLite requires `LIMIT` to be present with `OFFSET`; `BookStore.find()` didn't guard this. Now emits `LIMIT -1` (SQLite's "no limit") when offset is given without limit.
- **Fixed dead `lastInsertRowid` guard** ‚Äî `result.lastInsertRowid !== undefined` was always true in better-sqlite3; removed and updated type comment to explain actual SQLite semantics.
- **Wrote 59 unit tests** across three new files (`sqlite-adapter.test.ts`, `book-store.test.ts`, `reconcile-books.test.ts`). All pass. Tests cover CRUD, filtering, ordering, pagination, injection guards, normalization, idempotency.

## Decisions made and/or deferred

- **`/` ‚Üí `__`, not `_`** ‚Äî simple `_` substitution collapses the scope separator and produces an injective-enough mapping; `__` preserves the structural distinction. Theoretical collision (`acme/my-rig` vs `acme/my_rig`) judged unreachable in practice.
- **Normalize rig keys, validate book names** ‚Äî rig keys are opaque, framework-derived identifiers so normalization is appropriate. Book names are author-declared and should already be safe; throw loudly if not.
- **`validateFieldName` exported, `normalizeRigKey` not** ‚Äî `validateFieldName` is shared internal infrastructure; `normalizeRigKey` is a `booksTableName` implementation detail.
- **No live guilds** ‚Äî migration concern for renamed tables was a non-issue.
