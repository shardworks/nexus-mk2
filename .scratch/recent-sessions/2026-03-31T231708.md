## Decisions made and/or deferred

- **Apparatus name: "The Stacks"**, plugin id `stacks`, package `@shardworks/stacks`. Dropping the `nexus-` prefix convention for standard plugins.
- **`prev` always provided** for update/delete CDC events. Pre-read cost accepted for cascade use cases.
- **`failOnError` determines execution phase**, not just error handling: `true` = Phase 1 (inside transaction), `false` = Phase 2 (after commit).
- **Transactions in scope.** Every write opens an implicit transaction spanning the write + all Phase 1 CDC handlers. Explicit `transaction()` available for caller-initiated grouping.
- **CDC events coalesced per-document** on transaction commit. Six coalescing rules (create→update = create, create→delete = no event, etc.).
- **OR conditions permanently out of scope** — run two queries, merge in app code.
- **Backend swappability via apparatus replacement**, not sub-apparatus pattern. Deferred until there's a second backend.
- **Deferred:** declarative `watches` on kit exports, deep-merge patch, batch operations, raw SQL read escape hatch.
