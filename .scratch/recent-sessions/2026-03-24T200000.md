## What we did

- **Refactored `nexus init` to bootstrap via `installTool`** — `initGuild()` is now skeleton-only (bare repo, worktree, dirs, empty guild.json, migration file, commit). New `bootstrapBaseTools()` installs all framework tools via `installTool({ framework: true })`, using actual package directories as source. CLI orchestrates: `initGuild` → `bootstrapBaseTools` → `applyMigrations`. Committed as `e36d947`.
- **Extended `installTool`** with `framework` option (installs to `nexus/` prefix, registers as `source: 'nexus'`) and `commit` option (for batching during bootstrap).
- **Slimmed `base-tools.ts`** — replaced 200+ lines of embedded instruction templates with simple `{ name, packageName, roles? }` refs. Packages themselves are the source of truth.
- **Fixed migration engine PRAGMA handling** — PRAGMAs now extracted and run outside transactions (SQLite can't change journal_mode inside a transaction).
- **Fixed `copyDir`** to skip `node_modules`/`.git` and handle symlinks when copying from real package directories.

## Decisions made and/or deferred

- **`installTool` is the universal install path** — framework and guild tools use the same code. `framework: true` routes to `nexus/` dirs and sets `source: 'nexus'`.
- **Package directories are the source of truth** — no more duplicated instructions/descriptions embedded in core. The actual implement/engine packages contain their own descriptors.
- **Package resolution injected by caller** — `bootstrapBaseTools` takes a `resolvePackage` callback rather than importing all packages. Keeps core decoupled from module resolution.
- **Single bootstrap commit** — all base tools installed without individual commits, one "Bootstrap base tools" commit at the end.
