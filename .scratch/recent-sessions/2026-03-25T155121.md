# Session Summary

## What we did

- **Designed CLI restructure around `nsg <noun> <verb>` pattern** — drafted a comprehensive spec at `.scratch/cli-restructure-spec.md`. Covers every noun group (workshop, tool, anima, commission, clock, work, piece, job, stroke), standard verbs (create/list/show/update/remove), top-level exceptions (init, status, consult, signal), and auto-wiring architecture between CLI commands and MCP tools.
- **Established `<noun>-<verb>` tool naming convention** — every CLI noun-verb command maps 1:1 to a tool. ~37 tools total (up from 6). All tools registered; guild controls per-role surface via `guild.json`.
- **Decided on entity ID format** — lowercase prefixed random hex, 8 characters everywhere. `w-a3f7b2c1`, `j-7c2b1a09`, `c-b1a2c3d4`, `a-5e6f7a8b`, `evt-d8e2f1a0`, `ses-4b7c9e01`. No auto-increment IDs anywhere. Generated via `crypto.randomBytes(4).toString('hex')`.
- **Renamed `workshop add` → `workshop register`** for clarity vs. `workshop create`.
- **Resolved: no `remove` for work items** — work/piece/job/stroke are historical records; cancellation is a status transition via `update`.

## Decisions made and/or deferred

- **All entities get hex IDs, including clock events and sessions.** Sean's reasoning: auto-increment numbers carry false weight ("event #7" sounds foundational) and invite gap-reasoning. Hex IDs are inert identifiers where ordering comes from timestamps.
- **`commission` gets noun group + bare alias.** `nsg commission "spec"` is shorthand for `nsg commission create "spec"`.
- **`anima manifest` stays as a special verb** — it's a debug/inspect tool, not CRUD, not useful as an agent tool.
- **Read-only tools (list/show) are real registered tools.** Agents shouldn't need raw SQL. Guild policy controls which roles see which tools.
- **`update` handles status transitions** — no separate `set-status` verb. If status transitions need richer semantics later, add as a specialized alias.
- **Deferred: status lifecycles for work items** — out of scope, belongs in Ledger schema design.
- **Deferred: actual implementation** — spec is designed to be commissionable in phases.

### Notable moments

This reframed IDs as carrying implicit narrative weight — sequential numbers tell a story about ordering and importance that random identifiers don't. Applied universally, not just to human-facing entities.
- **Sean on `update` vs. `set-status`:** Asked "should 'set-status' be its own verb?" then let Coco analyze the tradeoff. Accepted the single-verb recommendation with the alias escape hatch.
- **Sean cut scope cleanly:** "decompose/plan/etc. are not things i want. just the create/list/show" — removed domain-operation verbs from the CLI, keeping it pure CRUD + status transitions. The domain operations (decomposition, planning) happen through events and agent sessions, not CLI commands.
