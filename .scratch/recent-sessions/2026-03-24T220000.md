# Session Summary

## What we did

- **Designed and implemented npm-based tool installation** — made the guildhall an npm package so guild tools can have third-party dependencies. `installTool` now supports npm registry, local directory (via npm install or symlink), and tarball sources. Committed as `f675e06`.
- **Added guild name** to `GuildConfig` and `initGuild` — guildhall `package.json` uses it. CLI passes name through from init prompt.
- **Updated manifest engine** — `McpServerConfig` now includes `NODE_PATH` pointing to the guildhall's `node_modules` so the MCP server can resolve guild tool imports.
- **Updated removeTool** — runs `npm uninstall` for npm-installed tools, removes symlinks for linked tools.
- **Added `package` field injection at install time** — `installTool` reads the source's `package.json` name and writes it into the slot descriptor. Removed manually-added `package` fields from all 10 framework descriptors.
- **Rewrote `docs/guides/building-implements.md`** — covers all install types, dependency resolution, `tsconfig.json` not required for guild tools.
- **Designed layout refactor** — NEXUS_HOME becomes a regular git clone (no more bare repo + worktree). Workshop repos and worktrees move to `.nexus/`. Removed `NEXUS_HOME` env var in favor of `--guild-root` flag + cwd-based guild discovery. Full work plan at `.scratch/guild-layout-refactor.md`.
- **Designed revised install types** — five types: registry, git-url, workshop (`workshop:forge#ref`), tarball, link. Local directory installs removed. Work plan includes durability analysis for each type.

## Decisions made and/or deferred

- **Guildhall is an npm package** — `package.json` at guildhall root, `node_modules` gitignored. Guild tools installed as npm dependencies.
- **`package` field derived at install time** — not stored in source descriptors. `installTool` reads from `package.json` name and writes to slot descriptor.
- **No more local directory installs** — forge agents use workshop installs (`workshop:name#ref`). Dev iteration uses `--link`.
- **Workshop installs use `--no-save` + full source in slot** — avoids machine-specific `git+file://` paths in `package.json`. Slot is a durable snapshot.
- **NEXUS_HOME env var removed** — replaced by `--guild-root` flag + walking up from cwd to find `guild.json`.
- **NEXUS_HOME IS the guildhall** — regular git clone, not bare repo + worktree. `.nexus/` holds workshops, worktrees, and Ledger.
- **Deferred:** Actual implementation of layout refactor and revised install types — work plan written for another agent.

- **"Let's get rid of local directory installs entirely"** — Sean narrowing scope by eliminating a source type. The forge use case drove local installs, but workshop installs (`workshop:name#ref`) are a better fit. Notable as a "delete the feature" decision that simplifies the system.
