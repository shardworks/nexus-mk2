## Decisions made and/or deferred

- **Polling at 2000ms default** — boring, works. Configurable via `--interval`.
- **Serial processing only** — won't scale long-term, but prevents accidental token budget consumption from parallel anima sessions.
- **`tick` and `run` bail when daemon is running** — warn (not error), then exit without processing. No racing with the daemon.
- **Daemon script in core with dynamic import** — after two rounds of refactoring. Started with separate script in CLI (dependency issue), then tried hidden CLI subcommand (`_daemon`), then settled on daemon script in core using `@ts-expect-error` dynamic `import()` for the session provider. Self-contained, no parameters needed from callers.
- **Start/stop/status exposed as MCP tools** — steward can manage the daemon, not just check it.
- **Deferred:** Concurrency, log rotation, healthcheck, dashboard integration, webhook/external event injection.
