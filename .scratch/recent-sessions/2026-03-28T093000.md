## Decisions made and/or deferred

- **`Rig` as the author-facing export type** — chosen over "Framing A" (tools-only bare export). Rationale: `{ tools? }` gives flexibility for future lifecycle hooks (engines, init callbacks) without committing to them now.
- **`instance` not `definition`** — Sean preferred `instance` as the field name on `LoadedRig` for the raw module export. *"any reason not to use 'instance'? definition isn't my favorite here"*
- **`LoadedRig` not `ActiveRig` / `RuntimeRig`** — Sean picked `LoadedRig` from the offered options (LoadedRig, ActiveRig, RuntimeRig, "RigThatsJuicedUp 😄").
- **`migrations` removed from `RigDescriptor`** — Sean flagged it: *"those aren't going to be a thing"*. Removed without replacement.
- **Legacy/1 shims** — decided to leave backward-compat aliases in `legacy/1/plugin-descriptor.ts` for any legacy code importing old type names; new barrel (`core/src/index.ts`) exports new names only.
- **No `RigDescriptor` in `rig.json`** — kept as plain JSON read with `as RigDescriptor` cast; no runtime validation schema added (deferred).
