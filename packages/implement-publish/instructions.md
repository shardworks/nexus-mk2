# publish

Publish completed commission output to the guild — delivering finished work, updating the Ledger, and notifying relevant parties.

## Usage

```
publish <commissionId> [--summary <text>]
```

## Arguments

- `<commissionId>` — The ID of the commission whose output is being published
- `--summary <text>` — Brief summary of what was accomplished

## Guidance

- Only publish when the commission's acceptance criteria have been met.
- Include a clear summary so that other animas and leadership can understand what was delivered without reading the full output.
- Publishing is a terminal action for a commission — once published, the work is considered delivered.
